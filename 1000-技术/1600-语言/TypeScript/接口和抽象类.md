---
title: 接口和抽象类
---

## 接口

什么是接口：接口是一种规范的定义，它定义了行为和动作的规范。

### 接口的定义

```ts
interface Person {
  name: string;
  age: number;
}
```

### 接口的继承

```ts

interface Person {
  name: string;
  age: number;
}

interface Student extends Person {
  school: string;
}
```

### 接口的实现

```ts

interface Person {
  name: string;
  age: number;
}

class Student implements Person {
  name: string;
  age: number;
  school: string;
}
```

## 抽象类

什么是抽象类：抽象类是不允许被实例化的类，它可以包含抽象方法。

### 抽象类的定义

```ts
abstract class Person {
  name: string;
  age: number;
  abstract say(): void;
}
```

### 抽象类的继承

```ts
abstract class Person {
  name: string;
  age: number;
  abstract say(): void;
}

class Student extends Person {
  school: string;
  say() {
    console.log("say");
  }
}
```

### 抽象类的实现

```ts
abstract class Person {
  name: string;
  age: number;
  abstract say(): void;
}

class Student implements Person {
  name: string;
  age: number;
  school: string;
  say() {
    console.log("say");
  }
}
```

### 抽象类的构造函数

```ts
abstract class Person {
  name: string;
  age: number;
  abstract say(): void;
  constructor(name: string, age: number) {
    this.name = name;
    this.age = age;
  }
}

class Student extends Person {
  school: string;
  say() {
    console.log("say");
  }
}
```
